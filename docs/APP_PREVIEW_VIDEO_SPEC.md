# Ora AI App Preview Video Specification
**Task:** ORA-093 - Record 30-second app preview video  
**Due:** Pre-submission (optional but recommended)  
**Status:** Ready for production

## Overview

Create a compelling 30-second app preview video that showcases Ora AI's key features in action. This video auto-plays in the App Store (without sound) and is a powerful tool to convert browsers into downloaders.

**Goal:** Show the app experience, not tell. Let users see themselves using Ora AI.

## Technical Requirements

### Video Specifications
- **Duration:** 15-30 seconds (recommended: 30s for full story)
- **Format:** .mov or .mp4 (H.264 codec)
- **Resolution:** 1080 x 1920 (portrait, 9:16 aspect ratio)
- **Frame Rate:** 30 fps minimum
- **File Size:** <500 MB (App Store limit)
- **Audio:** Optional music track (royalty-free)

### Quality Requirements
- **Bitrate:** 8-10 Mbps minimum
- **Color Space:** sRGB
- **No Watermarks:** Clean, professional footage only

## Visual Style

### Transitions
- **Between scenes:** Smooth cross-fades or iOS native swipes
- **Within scenes:** Actual app navigation animations
- **No jarring cuts:** Maintain flow and calm energy
- **Timing:** Let moments breathe, don't rush

### Text Overlays
- **Font:** Sentient (brand font) for headlines
- **Size:** Large enough to read on mobile
- **Color:** White with subtle shadow or dark overlay background
- **Animation:** Fade in, pause, fade out
- **Duration:** 2-3 seconds per text card

### Pacing
- Calm and intentional (not fast-paced)
- Matches wellness app vibe
- Gives users time to absorb each feature
- Music guides the emotional tone

## Audio

### Background Music
**Style:** Calm, uplifting, minimal

**Options:**
- Piano + ambient pads
- Acoustic guitar with light percussion
- Soft electronic with warm tones
- Nature sounds (subtle)

**Volume:** Background level, doesn't overpower
**Tempo:** 80-100 BPM (calm, not energetic)

**Sources for Royalty-Free Music:**

**Selected Track Example:** Search for "peaceful ambient piano wellness app"

### No Voiceover
- App Store videos auto-play muted
- Text overlays communicate the message
- Music provides emotional tone

## Production Process

### Pre-Production
1. **Finalize script/storyboard** (this document)
2. **Prepare app** - Remove test data, polish animations
3. **Practice navigation** - Know exactly where to tap
4. **Set up recording** - Test screen recording quality

### Recording

**Method 1: Screen Recording (iOS Simulator)**
- Use Xcode Simulator → File → Record Screen
- Navigate through app following storyboard
- Capture multiple takes for each scene
- Resolution: 1080 x 1920 (portrait)

**Method 2: Physical Device Recording**
- Connect iPhone via USB
- Open QuickTime Player → New Movie Recording
- Select iPhone as camera source
- Record screen directly

**Tips:**
- Clean device (no notifications, full battery icon)
- Hide status bar if possible
- Smooth, deliberate gestures
- Capture 2-3 takes of each scene for options

### Post-Production (Editing)

**Tools:**
- **Final Cut Pro** (Mac, professional)
- **Adobe Premiere Pro** (cross-platform, professional)
- **iMovie** (Mac, free, good enough)
- **CapCut** (mobile-friendly, free)

**Editing Steps:**
1. Import all screen recording clips
2. Arrange clips according to storyboard
3. Trim to best takes
4. Add transitions (cross-fades, 0.5-1s duration)
5. Add text overlays (Sentient font, white with shadow)
6. Add background music track
7. Adjust audio levels (music should be background)
8. Color grade for consistency (if needed)
9. Add final title card ("Download Ora AI")
10. Export at correct specs

### Export Settings

**Resolution:** 1080 x 1920 (portrait)  
**Frame Rate:** 30 fps  
**Codec:** H.264  
**Bitrate:** 8-10 Mbps
